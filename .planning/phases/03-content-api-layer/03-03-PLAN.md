Define TypeScript interface `ContatoInput`: { nome: string, email: string, assunto: 'Dúvida' | 'Sugestão' | 'Parceria' | 'Outro', mensagem: string }.

- `createContato(data: ContatoInput): Promise<{ success: true } | { success: false; error: string }>` — insert into contatos table. Store ip_hash as empty string '' for now (rate limiting in Route Handler uses Redis, not DB hash). On Supabase error, log details server-side and return { success: false, error: 'Erro interno ao salvar mensagem' } (SEC-08: never expose DB error to client).

- `getContatos(): Promise<ContatoInput[] | { success: false; error: string }>` — admin-only read. SEC-09: enforce a RUNTIME auth double-verification guard inside this function (not just a JSDoc comment) — callers in Phase 5 may forget the JSDoc, but cannot bypass an in-function check. Implement as:

```typescript
export async function getContatos(): Promise<ContatoInput[] | { success: false; error: string }> {
  const supabase = await createClient()

  // SEC-09: runtime admin verification — double-verification pattern
  const { data: { user }, error: authError } = await supabase.auth.getUser()
  if (authError || !user) {
    return { success: false, error: 'Não autenticado' }
  }

  const { data: profile, error: profileError } = await supabase
    .from('profiles')
    .select('role')
    .eq('id', user.id)
    .single()

  if (profileError || !profile || profile.role !== 'admin') {
    return { success: false, error: 'Acesso negado — requer perfil admin' }
  }

  const { data, error } = await supabase
    .from('contatos')
    .select('*')
    .order('created_at', { ascending: false })

  if (error) {
    console.error('[getContatos]', error)
    return { success: false, error: 'Erro ao buscar contatos' }
  }

  return data ?? []
}
```

This enforces SEC-09 independently of any Route Handler or Server Action that calls it — the auth check cannot be skipped by future implementors.

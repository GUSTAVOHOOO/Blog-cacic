Install required packages first: `npm install gray-matter next-mdx-remote` (gray-matter for frontmatter parsing; next-mdx-remote v4 for App Router RSC compatibility — NOT @next/mdx which requires webpack plugin and doesn't support runtime filesystem reads).

Create `src/lib/mdx.ts` using Node.js `fs` and `path` modules + `gray-matter`:

```typescript
import 'server-only'
import fs from 'fs'
import path from 'path'
import matter from 'gray-matter'
import type { PostFrontmatter, NoticiaFrontmatter, PostMeta, NoticiaMeta } from '@/types/content'

const POSTS_DIR = path.join(process.cwd(), 'content/posts')
const NOTICIAS_DIR = path.join(process.cwd(), 'content/noticias')

function computeReadingTime(content: string): number {
  const words = content.trim().split(/\s+/).length
  return Math.ceil(words / 200)
}
```

Implement getAllPosts(): read POSTS_DIR with fs.readdirSync (return [] if directory missing), filter `.mdx` files, parse each with gray-matter, cast data as PostFrontmatter, filter published === true, add readingTime, sort by date descending.

Implement getPostBySlug(slug): find the file where frontmatter.slug === slug (loop over all files), return { frontmatter, source: content } or null if not found.

Implement getAllNoticias() and getNoticiaBySlug() identically for NOTICIAS_DIR using NoticiaFrontmatter / NoticiaMeta types.

IMPORTANT: `import 'server-only'` at top of file — this module reads the filesystem and must never run in the browser.

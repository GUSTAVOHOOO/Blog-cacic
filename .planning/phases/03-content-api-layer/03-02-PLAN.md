  <name>Task 3: MDXRenderer — Server Component wrapper for MDXRemote (no HTML sanitization)</name>
  <files>src/components/mdx/MDXRenderer.tsx</files>
  <action>
Create `src/components/mdx/MDXRenderer.tsx` as a Server Component (no 'use client' — rendering MDX in RSC is the correct pattern with next-mdx-remote v4).

DO NOT install or use isomorphic-dompurify or any DOMPurify package here. Reason: MDX source strings come from the filesystem (content/posts/, content/noticias/) — they are trusted author content, not user-submitted HTML. Applying an HTML tag allowlist with DOMPurify.sanitize() before MDX compilation would silently strip all custom component invocations (Note, Warning, Image, and any future custom tags) because DOMPurify does not know they are MDX JSX elements. SEC-06 XSS protection is provided by the Content Security Policy headers already configured in next.config.js, not by pre-processing MDX source.

```typescript
import { MDXRemote, MDXRemoteProps } from 'next-mdx-remote/rsc'
import { mdxComponents } from './MDXComponents'

interface MDXRendererProps {
  source: string
  components?: MDXRemoteProps['components']
}

export async function MDXRenderer({ source, components }: MDXRendererProps) {
  // MDX source is read from the filesystem (content/) — trusted author content.
  // No HTML sanitization here: DOMPurify's tag allowlist would silently strip
  // custom MDX component tags (Note, Warning, Image) before MDXRemote compiles them.
  // XSS protection is handled by CSP headers in next.config.js (SEC-06).
  return (
    <MDXRemote
      source={source}
      components={{ ...mdxComponents, ...components }}
      options={{
        mdxOptions: {
          // rehype-pretty-code for syntax highlighting is added here in Phase 4
          // when MDXRenderer is connected to page components
        },
      }}
    />
  )
}
```

Export as named export `MDXRenderer`. Phase 4 page components will import MDXRenderer and pass the `source` string returned by getPostBySlug()/getNoticiaBySlug(). Phase 4 will also add `rehypePrettyCode` to `mdxOptions.rehypePlugins` here to enable token coloring.

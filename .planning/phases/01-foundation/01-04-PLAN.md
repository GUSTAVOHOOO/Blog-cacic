  <how-to-verify>
    **Step 1: Push to GitHub**
    1. Create a new repository on GitHub (e.g., `blog-cacic`)
    2. Run from the project directory:
       ```bash
       git remote add origin https://github.com/YOUR_USERNAME/blog-cacic.git
       git branch -M main
       git push -u origin main
       ```

    **Step 2: Deploy to Vercel**
    1. Go to https://vercel.com/new
    2. Click "Import Git Repository" and select your `blog-cacic` repo
    3. Vercel will detect Next.js automatically (or use vercel.json settings)
    4. Before clicking "Deploy", click "Environment Variables" and add ALL of these:
       - `NEXT_PUBLIC_SUPABASE_URL` = your Supabase project URL
       - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = your Supabase anon key
       - `SUPABASE_SERVICE_ROLE_KEY` = your Supabase service role key
       - `NEXT_PUBLIC_APP_URL` = (set this AFTER first deploy with the actual Vercel URL)
       - `UPSTASH_REDIS_REST_URL` = (leave empty for now — needed in Phase 3)
       - `UPSTASH_REDIS_REST_TOKEN` = (leave empty for now — needed in Phase 3)
    5. Click "Deploy"

    **Step 3: Verify deployment**
    1. Wait for build to complete (green checkmark)
    2. Open the live URL (e.g., https://blog-cacic.vercel.app)
    3. Confirm you see "Blog CACIC — em construção"
    4. Open browser devtools -> Network -> click the main request -> Response Headers
    5. Confirm these headers are present:
       - `strict-transport-security`
       - `x-frame-options: DENY`
       - `x-content-type-options: nosniff`
    6. Visit `[YOUR_VERCEL_URL]/api/health` — should return `{"status":"ok"}` or `{"status":"degraded"}`
    7. Go back to Vercel -> Settings -> Environment Variables and add:
       - `NEXT_PUBLIC_APP_URL` = your actual Vercel URL (e.g., https://blog-cacic.vercel.app)
    8. Trigger a redeploy (Vercel -> Deployments -> Redeploy)

    **Step 4: Run security header check (optional but recommended)**
    Visit https://securityheaders.com and enter your Vercel URL.
    Aim for at least a B rating. An A rating requires HSTS preloading (not required for Phase 1).

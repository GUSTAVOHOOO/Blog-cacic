    The SQL must include ALL of the following in order:

    **1. Enable extensions:**
    ```sql
    CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
    ```

    **2. Table: profiles**
    - `id` uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE
    - `username` text UNIQUE NOT NULL
    - `full_name` text
    - `avatar_url` text
    - `role` text NOT NULL DEFAULT 'member' CHECK (role IN ('admin', 'editor', 'member'))
    - `bio` text
    - `updated_at` timestamptz DEFAULT now()
    - Index on `username`
    - Trigger: `update_updated_at` — sets `updated_at = now()` on every UPDATE

    **3. Table: eventos**
    - `id` uuid PRIMARY KEY DEFAULT uuid_generate_v4()
    - `title` text NOT NULL
    - `slug` text UNIQUE NOT NULL
    - `description` text
    - `tipo` text NOT NULL CHECK (tipo IN ('palestra','workshop','hackathon','social','outro'))
    - `data_inicio` timestamptz NOT NULL
    - `data_fim` timestamptz
    - `local` text
    - `link_inscricao` text
    - `thumbnail_url` text
    - `publicado` boolean NOT NULL DEFAULT false
    - `created_by` uuid REFERENCES profiles(id) ON DELETE SET NULL
    - `created_at` timestamptz DEFAULT now()
    - Indexes on `data_inicio`, `tipo`, `publicado`

    **4. Table: membros_diretoria**
    - `id` uuid PRIMARY KEY DEFAULT uuid_generate_v4()
    - `nome` text NOT NULL
    - `cargo` text NOT NULL
    - `foto_url` text
    - `linkedin_url` text
    - `github_url` text
    - `gestao` int NOT NULL
    - `ativo` boolean NOT NULL DEFAULT true
    - `ordem` int NOT NULL DEFAULT 0
    - Indexes on `gestao`, `ativo`

    **5. Table: trabalhos_academicos**
    - `id` uuid PRIMARY KEY DEFAULT uuid_generate_v4()
    - `title` text NOT NULL
    - `slug` text UNIQUE NOT NULL
    - `authors` text[] NOT NULL DEFAULT '{}'
    - `area` text NOT NULL
    - `summary` text
    - `pdf_url` text
    - `thumbnail_url` text
    - `publicado` boolean NOT NULL DEFAULT false
    - `ano` int NOT NULL
    - `created_by` uuid REFERENCES profiles(id) ON DELETE SET NULL
    - `created_at` timestamptz DEFAULT now()
    - Full-text search index (Portuguese): `CREATE INDEX idx_trabalhos_fts ON trabalhos_academicos USING GIN (to_tsvector('portuguese', coalesce(title,'') || ' ' || coalesce(summary,'') || ' ' || coalesce(area,'')))`

    **6. Table: contatos**
    - `id` uuid PRIMARY KEY DEFAULT uuid_generate_v4()
    - `nome` text NOT NULL
    - `email` text NOT NULL
    - `assunto` text NOT NULL
    - `mensagem` text NOT NULL
    - `ip_hash` text — SHA-256 hash of IP, never raw IP
    - `created_at` timestamptz DEFAULT now()

    **7. Table: post_views**
    - `id` uuid PRIMARY KEY DEFAULT uuid_generate_v4()
    - `slug` text NOT NULL
    - `created_at` timestamptz DEFAULT now()
    - Index on `slug`

    **8. View: post_view_counts**
    ```sql
    CREATE VIEW post_view_counts AS
      SELECT slug, COUNT(*) AS views
      FROM post_views
      GROUP BY slug;
    ```

    **9. Trigger function for updated_at:**
    ```sql
    CREATE OR REPLACE FUNCTION handle_updated_at()
    RETURNS TRIGGER AS $$
    BEGIN
      NEW.updated_at = now();
      RETURN NEW;
    END;
    $$ LANGUAGE plpgsql;

    CREATE TRIGGER update_profiles_updated_at
      BEFORE UPDATE ON profiles
      FOR EACH ROW EXECUTE FUNCTION handle_updated_at();
    ```

    **10. RLS — Enable on all tables:**
    ```sql
    ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
    ALTER TABLE eventos ENABLE ROW LEVEL SECURITY;
    ALTER TABLE membros_diretoria ENABLE ROW LEVEL SECURITY;
    ALTER TABLE trabalhos_academicos ENABLE ROW LEVEL SECURITY;
    ALTER TABLE contatos ENABLE ROW LEVEL SECURITY;
    ALTER TABLE post_views ENABLE ROW LEVEL SECURITY;
    ```

    **11. RLS Policies — profiles:**
    - SELECT: authenticated users can read all profiles; anon can read basic info (no bio/email)
    - INSERT: users can only insert their own profile (auth.uid() = id)
    - UPDATE: users can only update their own profile (auth.uid() = id)
    - DELETE: admin only (check role in profiles via service role — leave a comment that this is handled by service_role in API)

    Write realistic policies:
    ```sql
    -- profiles: anyone can read
    CREATE POLICY "profiles_select" ON profiles FOR SELECT USING (true);
    -- profiles: users insert own row
    CREATE POLICY "profiles_insert_own" ON profiles FOR INSERT WITH CHECK (auth.uid() = id);
    -- profiles: users update own row
    CREATE POLICY "profiles_update_own" ON profiles FOR UPDATE USING (auth.uid() = id);
    ```

    **12. RLS Policies — eventos:**
    ```sql
    -- Public can read published events
    CREATE POLICY "eventos_select_published" ON eventos FOR SELECT USING (publicado = true);
    -- Authenticated editors/admins can read all (including drafts)
    CREATE POLICY "eventos_select_auth" ON eventos FOR SELECT USING (
      auth.role() = 'authenticated' AND EXISTS (
        SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('editor','admin')
      )
    );
    -- Editors/admins can insert
    CREATE POLICY "eventos_insert_editor" ON eventos FOR INSERT WITH CHECK (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('editor','admin'))
    );
    -- Editors/admins can update their own; admins can update any
    CREATE POLICY "eventos_update_editor" ON eventos FOR UPDATE USING (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role IN ('editor','admin'))
    );
    -- Admins can delete
    CREATE POLICY "eventos_delete_admin" ON eventos FOR DELETE USING (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
    );
    ```

    **13. RLS Policies — membros_diretoria:**
    ```sql
    -- Public read
    CREATE POLICY "membros_select" ON membros_diretoria FOR SELECT USING (true);
    -- Admin write
    CREATE POLICY "membros_insert_admin" ON membros_diretoria FOR INSERT WITH CHECK (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
    );
    CREATE POLICY "membros_update_admin" ON membros_diretoria FOR UPDATE USING (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
    );
    CREATE POLICY "membros_delete_admin" ON membros_diretoria FOR DELETE USING (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
    );
    ```

    **14. RLS Policies — trabalhos_academicos:**
    ```sql
    -- Public read published
    CREATE POLICY "trabalhos_select_published" ON trabalhos_academicos FOR SELECT USING (publicado = true);
    -- Authenticated read all
    CREATE POLICY "trabalhos_select_auth" ON trabalhos_academicos FOR SELECT USING (auth.role() = 'authenticated');
    -- Members can insert
    CREATE POLICY "trabalhos_insert_member" ON trabalhos_academicos FOR INSERT WITH CHECK (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid())
    );
    -- Owners and admins can update
    CREATE POLICY "trabalhos_update_owner" ON trabalhos_academicos FOR UPDATE USING (
      auth.uid() = created_by OR
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
    );
    -- Admins can delete
    CREATE POLICY "trabalhos_delete_admin" ON trabalhos_academicos FOR DELETE USING (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
    );
    ```

    **15. RLS Policies — contatos:**
    ```sql
    -- Only admins can read contact messages
    CREATE POLICY "contatos_select_admin" ON contatos FOR SELECT USING (
      EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin')
    );
    -- Anyone (anon) can insert (submit contact form) — rate limiting handled at API level
    CREATE POLICY "contatos_insert_anon" ON contatos FOR INSERT WITH CHECK (true);
    ```

    **16. RLS Policies — post_views:**
    ```sql
    -- Anyone can insert a view (anonymous tracking)
    CREATE POLICY "post_views_insert_anon" ON post_views FOR INSERT WITH CHECK (true);
    -- Anyone can read view counts
    CREATE POLICY "post_views_select" ON post_views FOR SELECT USING (true);
    ```
